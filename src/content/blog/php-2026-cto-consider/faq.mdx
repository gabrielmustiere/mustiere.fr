questions:
  - q: 'Should you really start a new project in PHP/Symfony rather than Node or Python in 2026?'
    r: |
      It depends on the product. For a web app that consumes LLMs through APIs and needs API + back-office + e-commerce + mobile coverage, Symfony is today
      the fastest choice to get up and running — API Platform, EasyAdmin, Sylius, Live Components and the Hotwire Native bridge all come out of the box,
      with no glue code. For a strongly ML-oriented product (training, custom inference, large-scale data processing), Python remains better placed. For a
      strongly real-time service under one millisecond or systems tooling, Go or Rust come out ahead. The trap is reasoning by genre rather than by use case.
  - q: 'FrankenPHP in worker mode: what does the migration cost on an existing Symfony codebase?'
    r: |
      Low, in most cases. The rule fits in one sentence: don't keep mutable state between two requests. The rare services that cache request-scoped data
      must implement `ResetInterface`, which is quickly fixed. Once that step is past, the gain is immediate: on the Symfony 7.4 benchmark published in
      early 2026, you go from 1,240 RPS / p95 45 ms (Nginx + PHP-FPM) to 3,850 RPS / p95 8 ms (FrankenPHP worker). Deployment is simplified too — single
      binary, integrated Caddy, automatic HTTPS, native HTTP/3 and Mercure. RoadRunner and Swoole remain serious alternatives for those who have already
      invested in them.
  - q: 'How long for a Node or Python team to become productive in Symfony?'
    r: |
      On the teams I've seen migrate, a senior developer comfortable with a typed full-stack framework (Spring, Rails, NestJS, Django) reaches a decent
      level in two to four weeks, and full productivity in two to three months. The main brake isn't the language — PHP 8.5 is close to a modern C# or
      Kotlin — but getting familiar with Symfony conventions (DI, services, Doctrine, Messenger). It's a modest investment compared to the functional
      coverage you get back in return.
